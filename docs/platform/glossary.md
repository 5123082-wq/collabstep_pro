# Glossary

**Статус:** active  
**Владелец:** product  
**Последнее обновление:** 2026-01-06

Используйте этот файл как **канонический словарь** терминов платформы и документации.

## Основные сущности

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Workspace** | Рабочее пространство внутри организации. Содержит проекты и команду. | Синоним "Account". Пользователь может иметь доступ к нескольким workspace в разных организациях. |
| **Organization** | Организация (компания, команда). Может содержать несколько workspace. | Поддержка мультиаккаунта: пользователь может состоять в нескольких организациях. |
| **Project** | Проект, принадлежащий workspace. Содержит задачи, файлы, документы, команду. | Проект может быть создан из шаблона (`ProjectTemplate`). |
| **Task** | Задача в проекте. Поддерживает иерархию (родитель-ребенок), статусы, метки, назначения. | Задачи могут иметь зависимости (`TaskDependency`), комментарии (`TaskComment`), файлы. |
| **Project Template** | Шаблон проекта для создания новых проектов. Может содержать задачи-шаблоны. | Шаблоны могут быть публичными (маркетплейс) или приватными (workspace). |
| **Project Template Task** | Задача-шаблон в шаблоне проекта. Определяет структуру задач при создании проекта из шаблона. | Поддерживает иерархию, дефолтные статусы, метки, смещения дат. |

## Модули

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **PM Core** | Основной модуль управления проектами. Включает проекты, задачи, команды, представления. | Kanban, List, Calendar, Gantt, Dashboard представления. |
| **Marketplace** | Маркетплейс продуктов и услуг. Публикация проектов как шаблонов, каталог, корзина, заказы. | Продукты продаются через корзину, услуги через запрос (inquiry). |
| **Marketing Dashboards** | Маркетинговая аналитика и интеграции. ROI/CPC/CPA/CLV метрики, кампании, отчеты. | Интеграции с Google Ads, Facebook Ads (планируется). |
| **AI Hub** | Модуль AI-интеграции. Ассистенты, генерация контента, промпты, агенты. | Поддержка OpenAI, Yandex GPT и других провайдеров. |
| **Finance** | Финансовый модуль. Бюджеты проектов, расходы, отчетность. | Интеграция с PM Core для отслеживания расходов по проектам. |
| **Community** | Модуль сообщества. Питчи, комнаты, взаимодействие. | В разработке. |
| **Performers** | Модуль исполнителей. Специалисты, команды, вакансии. | В разработке. |

## Marketplace

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Listing** | Публикация проекта в маркетплейсе. Может быть в статусе draft/published/rejected. | Создается из проекта через мастер публикации. |
| **Product** | Продукт в маркетплейсе. Продается через корзину (cart). | Шаблон проекта, опубликованный как продукт. |
| **Service** | Услуга в маркетплейсе. Предоставляется через запрос (inquiry/request). | Услуга требует согласования перед началом работы. |
| **Order** | Заказ в маркетплейсе. Создается при покупке продукта или запросе услуги. | Содержит информацию о покупателе, продукте/услуге, статусе, оплате. |
| **Cart** | Корзина покупок. Временное хранилище выбранных продуктов перед checkout. | Состояние корзины хранится в Zustand store. |

## PM Core

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Kanban** | Представление задач в виде колонок (статусов) с drag & drop. | Поддерживает группировки: assignee, priority, сроки. |
| **List** | Табличное представление задач (Excel-подобное). Настраиваемые колонки, инлайн-редактирование. | Экспорт в CSV, фильтры и сортировки на уровне колонок. |
| **Calendar** | Календарное представление задач. День/Неделя/Месяц, drag-to-reschedule. | Переключатель "Только мои/Все" задачи. |
| **Gantt** | Диаграмма Ганта для визуализации задач по срокам и зависимостям. | Показывает зависимости между задачами, критические пути. |
| **Dashboard** | Дашборд с виджетами: Пульс, Прогресс, Нагрузка исполнителей, Финансы. | Все виджеты кликабельны и ведут к дрилдаунам. |
| **Workflow** | Набор статусов для проекта. Определяет колонки Kanban и возможные переходы. | Проект может иметь несколько workflow, один по умолчанию. |
| **Iteration** | Итерация (спринт) в проекте. Группирует задачи по временным периодам. | Имеет start_at и end_at даты. |
| **Task Comment** | Комментарий к задаче. Поддерживает упоминания, файлы, реакции. | Реализовано в Stage H. |
| **Notification** | Уведомление пользователю. Создается при событиях в проектах/задачах. | Реализовано в Stage I. |
| **Project Chat** | Командный чат проекта. Сообщения между участниками проекта. | Реализовано в Stage J. |

## Финансы

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Project Budget** | Бюджет проекта. Содержит валюту, общий лимит, лимиты по категориям, порог предупреждения. | Обновляется через API, отслеживает потраченные суммы. |
| **Expense** | Расход проекта. Привязан к проекту, имеет категорию, сумму, статус. | Статусы: pending, approved, rejected, paid. |
| **Expense Category** | Категория расхода. Определяет тип расхода (материалы, услуги, зарплата и т.д.). | Лимиты по категориям задаются в бюджете проекта. |

## Роли и права

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Organization Role** | Роль пользователя в организации. | `owner`, `admin`, `member`, `viewer` |
| **Project Role** | Роль пользователя в проекте. | `owner`, `manager`, `contributor`, `viewer` |
| **Platform Role** | Системная роль пользователя на платформе. | `productAdmin`, `featureAdmin`, `financeAdmin`, `moderator`, `betaTester`, `viewer` |
| **User Role** | Роль пользователя в UI (клиентская). | `FOUNDER`, `SPECIALIST`, `CONTRACTOR`, `PM`, `ADMIN`, `MODERATOR`, `OBSERVER` |

## Технические термины

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Feature Flag** | Флаг функциональности. Включает/выключает фичи без деплоя. | `NAV_V1`, `AUTH_DEV`, `FEATURE_PROJECTS_V1`, `PM_NAV_PROJECTS_AND_TASKS` |
| **WebSocket** | Протокол для real-time обновлений. Используется для синхронизации состояния между клиентами. | Реализовано в Stage L. |
| **Vercel Blob Storage** | Хранилище файлов Vercel. Используется для файлов проектов. | Интеграция через `@vercel/blob`. |
| **Drizzle ORM** | ORM для работы с PostgreSQL. Используется для миграций и запросов. | Канонические таблицы: `pm_projects`, `pm_tasks`, `organizations`, `users`. |
| **Cache-aside** | Паттерн кэширования. Чтение из кэша, при промахе — из БД, запись в кэш. | Используется для оптимизации чтения проектов и задач. |

## AI Hub

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **AI Assistant** | AI-ассистент для проектов. Помогает с задачами, генерацией контента, анализом. | Реализовано в Stage M. |
| **AI Agent** | Автономный AI-агент для автоматизации задач. | Планируется в Stage N. |
| **Prompt** | Промпт для AI. Шаблон запроса к AI-модели. | Управление промптами в AI Hub. |
| **Generation** | Генерация контента AI. Тексты, изображения, код. | Поддержка различных провайдеров (OpenAI, Yandex GPT). |

## Статусы и состояния

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Project Status** | Статус проекта. | `active`, `on_hold`, `completed`, `cancelled`, `archived` |
| **Task Status** | Статус задачи. Определяется workflow проекта. | Зависит от workflow, может быть `new`, `in_progress`, `done` и т.д. |
| **Expense Status** | Статус расхода. | `pending`, `approved`, `rejected`, `paid` |
| **Listing State** | Состояние публикации в маркетплейсе. | `draft`, `published`, `rejected` |
| **Project Visibility** | Видимость проекта. | `private`, `workspace`, `public` |

## Документы и файлы

| Термин | Определение | Примечания / примеры |
|--------|-------------|---------------------|
| **Document** | Документ проекта. Договор, спецификация, отчет. | Поддерживает версионность через `DocumentVersion`. |
| **File** | Файл в проекте. Может быть привязан к задаче, документу, комментарию. | Хранится в Vercel Blob Storage, метаданные в БД. |
| **Folder** | Папка для организации файлов. Иерархическая структура. | Поддержка кастомных папок и системных (Results, Attachments). |
