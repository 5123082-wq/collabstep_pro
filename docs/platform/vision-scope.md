# Vision & Scope

**Статус:** active  
**Владелец:** product  
**Последнее обновление:** 2026-01-06

## Vision

Успех платформы Collabverse выглядит так:

- **Единая платформа** для управления проектами, маркетплейсом и маркетинговой аналитикой, объединяющая все инструменты в одном месте
- **Модульная архитектура** позволяет командам использовать только нужные модули (PM Core, Marketplace, Marketing, AI Hub)
- **Real-time коллаборация** через WebSocket обеспечивает синхронизацию состояния между участниками
- **AI-интеграция** помогает автоматизировать рутинные задачи и генерировать контент
- **Мультиаккаунт** позволяет пользователям работать в нескольких организациях и workspace

## Целевые аудитории

### B2B команды

- **Агентства** — управление клиентскими проектами, командами, бюджетами
- **Стартапы** — полный цикл разработки продукта от идеи до запуска
- **Компании** — внутренние проекты, команды, финансы

**Jobs-to-be-done:**
- Управление проектами с прозрачностью прогресса и бюджета
- Коллаборация команды в реальном времени
- Отслеживание финансов и расходов по проектам

### B2C профессионалы/креаторы

- **Фрилансеры** — управление проектами, клиентами, финансами
- **Креаторы** — публикация работ в маркетплейсе, управление заказами

**Jobs-to-be-done:**
- Публикация проектов как продуктов/услуг в маркетплейсе
- Управление заказами и клиентами
- Финансовый учет доходов и расходов

### Агентства

- **Дизайн-агентства** — управление проектами, командами, публикация портфолио
- **Маркетинговые агентства** — управление кампаниями, аналитика, отчеты

**Jobs-to-be-done:**
- Управление множеством клиентских проектов
- Публикация работ в маркетплейсе для привлечения клиентов
- Маркетинговая аналитика и отчеты для клиентов

## Core jobs-to-be-done (JTBD)

1. **Управление проектами и задачами**
   - Создание проектов из шаблонов или с нуля
   - Управление задачами с поддержкой иерархии, статусов, меток
   - Различные представления (Kanban, List, Calendar, Gantt) для разных рабочих стилей
   - Real-time синхронизация состояния между участниками

   - Приглашение участников в проекты и организации

   - Уведомления о важных событиях

3. **Финансовый контроль**
   - Бюджеты проектов с лимитами по категориям
   - Учет расходов с привязкой к проектам

   - Финансовая отчетность

4. **Публикация и продажа**
   - Публикация проектов как шаблонов в маркетплейсе
   - Продажа продуктов через корзину
   - Предоставление услуг через запросы
   - Управление заказами

5. **Маркетинговая аналитика**
   - Дашборды ROI/CPC/CPA/CLV
   - Интеграции с внешними платформами (Google Ads, Facebook Ads)
   - Экспорт и публикация отчетов

6. **AI-помощь**
   - AI-ассистент для проектов
   - Генерация контента (тексты, изображения)
   - Автоматизация рутинных задач через AI-агентов

## Non-goals / Out of scope

### Явные исключения

- **CRM для продаж** — не фокус платформы, хотя есть элементы (клиенты, заказы)
- **Бухгалтерский учет** — только управление бюджетами и расходами, не полный учет
- **Видео-конференции** — нет встроенных видеозвонков, только интеграции
- **Email-маркетинг** — нет встроенного email-маркетинга, только аналитика
- **Социальная сеть** — Community модуль для взаимодействия, не полноценная соцсеть
- **Мобильные приложения** — только веб-приложение, мобильные приложения не в scope

### Временные исключения

- **Полная интеграция с внешними платформами** — планируется, но не в MVP
- **Расширенная AI-функциональность** — базовая реализована, расширенная планируется
- **Мобильные приложения** — может быть рассмотрено в будущем

## Product principles

1. **Docs-as-code** — документация хранится в репозитории, версионируется, обновляется вместе с кодом
2. **Модульность** — каждый модуль независим, можно использовать отдельно
3. **Real-time** — синхронизация состояния в реальном времени через WebSocket
4. **Type safety** — строгая типизация TypeScript для надежности
5. **Feature flags** — управление функциональностью через флаги без деплоя
6. **Open architecture** — открытая архитектура для интеграций и расширений

## Success metrics (north star + supporting)

### North star metric

**Активные проекты с завершенными задачами** — количество проектов, в которых за период было завершено хотя бы одна задача.

Этот метрика отражает:
- Использование платформы для реальной работы
- Продуктивность команд
- Ценность платформы для пользователей

### Activation metrics

- **Регистрация → первый проект** — % пользователей, создавших первый проект в течение 7 дней
- **Первый проект → первая задача** — % пользователей, создавших первую задачу в течение 1 дня
- **Первая задача → завершенная задача** — % пользователей, завершивших первую задачу в течение 7 дней

### Retention metrics

- **DAU/MAU** — Daily/Monthly Active Users
- **Проекты на пользователя** — среднее количество активных проектов на пользователя
- **Задачи на проект** — среднее количество задач в проекте
- **Участники на проект** — среднее количество участников в проекте

### Revenue metrics

- **Подписки** — количество платных подписок (pro/max)
- **MRR** — Monthly Recurring Revenue
- **Конверсия free → paid** — % пользователей, перешедших на платную подписку
- **Marketplace GMV** — Gross Merchandise Value маркетплейса

### Engagement metrics

- **События на пользователя** — среднее количество событий (создание задач, комментарии, обновления) на пользователя
- **Время в приложении** — среднее время сессии
- **Частота использования** — количество дней использования в месяц
- **Модули на пользователя** — среднее количество используемых модулей (PM Core, Marketplace, Marketing, AI Hub)
