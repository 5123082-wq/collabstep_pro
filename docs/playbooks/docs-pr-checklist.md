# Docs PR Checklist

**Статус:** stable  
**Владелец:** product + engineering  
**Создан:** 2026-01-07  
**Последнее обновление:** 2026-03-08

## Назначение

Этот чеклист используется для PR, которые меняют документацию или реализуют функциональность. Его цель — гарантировать, что по репозиторию видно, что запланировано, что в работе, что завершено и что уже вышло.

## Чеклист Для PR С Реализацией Фич

### 1. Module Docs

- [ ] Обновлён основной документ модуля: `docs/modules/<module>/<module>-overview.md`
- [ ] Обновлены поддокументы модуля, если изменилась конкретная фича или flow
- [ ] Обновлён `docs/modules/<module>/<module>-implementation-plan.md`, если такой файл существует
- [ ] В `*-overview.md` синхронизированы `Purpose`, `Top user scenarios`, `Functional catalog`, `Implementation status`, `TODO / Future improvements` при необходимости

### 2. Plan / Status Tracking

- [ ] Обновлён `docs/ROADMAP.md`
- [ ] Для новых идей добавлена запись в `Текущие планы` или `Будущие улучшения`
- [ ] Для стартовавшей работы статус изменён на `🔄` и добавлена дата начала
- [ ] Для завершённой реализации статус изменён на `✅` и добавлена дата завершения
- [ ] Если работа поставлена на паузу, статус и оставшийся scope зафиксированы честно, без ложного `✅`

### 3. Active Working Context

- [ ] Обновлён `CONTINUITY.md`, если задача значимая, долгоживущая, прерывалась или влияет на handoff
- [ ] В `CONTINUITY.md` зафиксировано только активное состояние выше `STOP LINE`

### 4. Platform Docs

- [ ] Обновлён `docs/platform/overview.md`, если изменилась карта модулей или межмодульные связи
- [ ] Обновлён `docs/platform/glossary.md`, если появились новые термины
- [ ] Обновлён `docs/platform/roles-permissions.md`, если изменились права доступа
- [ ] Обновлён `docs/platform/analytics-events.md`, если появились или изменились события аналитики
- [ ] Обновлены `docs/README.md` и `docs/INDEX.md`, если появился новый модуль или новая значимая документационная entry point

### 5. Release History

- [ ] Обновлён `docs/platform/changelog.md`, если изменение реально вышло или оформляется в `Unreleased` как часть release-подготовки
- [ ] В changelog нет планов, незавершённых идей и рабочих заметок

### 6. Quality Checks

- [ ] Во всех изменённых документах обновлено поле `Последнее обновление`
- [ ] Актуальны статусы документов (`draft`, `review`, `stable`, `active`)
- [ ] Все внутренние ссылки валидны
- [ ] Markdown проходит принятые проверки
- [ ] Все неопределённости помечены через `NEEDS_CONFIRMATION`

## Чеклист Для PR Только С Документацией

- [ ] Обновлены правильные документы-источники истины, а не случайные дубликаты
- [ ] Если менялись статусы идей/этапов — обновлён `docs/ROADMAP.md`
- [ ] Если менялась структура модуля — обновлён соответствующий `*-overview.md`
- [ ] Если изменение относится к релизной истории — обновлён `docs/platform/changelog.md`
- [ ] Если правка влияет на handoff или текущую активную работу — обновлён `CONTINUITY.md`
- [ ] Архивные документы не использованы как актуальный источник без явной пометки

## Definition Of Done (Docs)

Фича или заметное изменение считаются документированными только если выполнены все применимые пункты:

- [ ] `docs/ROADMAP.md` отражает реальный статус
- [ ] `docs/modules/<module>/<module>-overview.md` отражает текущее состояние модуля
- [ ] Детализирующий `*-implementation-plan.md` синхронизирован, если он существует
- [ ] Cross-cutting docs обновлены при необходимости
- [ ] `CONTINUITY.md` отражает активное состояние работы, если это нужно для handoff
- [ ] `docs/platform/changelog.md` обновлён только для реально выпущенных изменений

## Легенда Статусов Этапов

- `✅ Завершён`
- `🔄 В работе`
- `⏳ Ожидает`
- `❌ Заблокирован`
- `⚠️ Частично`

## Быстрая Проверка Логики Документов

- `ROADMAP` отвечает на вопрос: что планируется, что в работе, что завершено?
- `module overview` отвечает на вопрос: что умеет модуль и в каком состоянии его функции?
- `implementation plan` отвечает на вопрос: какие этапы и зависимости у крупной реализации?
- `CONTINUITY` отвечает на вопрос: где мы остановились прямо сейчас?
- `changelog` отвечает на вопрос: что уже реально вышло?
