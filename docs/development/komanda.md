### 1. Договоримся о сущностях

Чтобы не путаться, разделим уровни:

- **Пользователь** – любой человек с аккаунтом (email + пароль).
- **Роль пользователя** – как он себя позиционирует:
  - **Организатор / основатель / менеджер** (создаёт организацию и проекты).
  - **Исполнитель** (дизайнер, разработчик, маркетолог и т.п.). 
((тоже может создавать организацию и проекты. Например дизайнер может создать проект в котором ему потребуется помощь маркетолога, либо программиста))
  - Можно разрешить **оба варианта** одновременно.
- **Организация / команда** – рабочее пространство (люди, проекты, задачи).
((организация может быть два вида «закрытая» «открытая». Открытая организация по сути это команда которая появляется в разделе Команды и подрядчики))
- **Исполнители (каталог)** – витрина специалистов, видимая другим.
- **Проект / задача** – единица работы, куда мы назначаем конкретных людей.

---

### 2. Регистрация: как человек «попадает» в систему

**Шаг 1. Базовая регистрация**

- Форма: email, пароль, имя.
- После этого – экран выбора роли:
((этот этап можно пропустить и выбрать потом))
  - **Вариант 1. «Я представляю бизнес / команду»**
  - **Вариант 2. «Я исполнитель (дизайнер, разработчик и т.д.)»**
  - Чекбокс: «Я хочу и то и другое» (опционально).

**Шаг 2. Если он выбирает «Я представляю бизнес»**

- Создаёт **организацию**:
  - Название компании.
  - Размер, индустрия (по желанию).
- Автоматически становится:
  - **Владелец организации**.
  - Попадает в раздел **«Организация → Команда»** как участник.

**Шаг 3. Если он выбирает «Я исполнитель»**

- Создаёт **профиль исполнителя**:
  - Специализация (дизайнер, frontend, backend и т.п.).
  - Навыки / стек / категория.
  - Краткое описание.
  - Часовая ставка / тип занятости (опционально).
  - Город / часовой пояс.
- Галочка: **«Показывать меня в каталоге исполнителей»**:
  - Включена → появляется в разделе **«Исполнители»** (общий каталог).
  - Отключена → профиль есть, но виден только тем, кто получил прямую ссылку или приглашение.

### 3. Как исполнитель попадает в раздел «Исполнители»

Логика:

- **Условие попадания в каталог**:
  - Есть выполненный профиль исполнителя (минимально заполнен).
  - Включён флаг «Показывать в каталоге».
  - (Опционально) прошёл модерацию / верификацию.

- **Где на интерфейсе**:
  - В личном кабинете исполнителя:
    - «Мой профиль» → «Профиль исполнителя»
    - Там переключатель: **«Показывать в каталоге»**.
  - Если он активен → карточка специалиста появляется в разделе **«Исполнители»**:
    - Имя, роль (дизайнер, разработчик), аватар.
    - Основные теги / навыки.
    - Кнопка **«Пригласить в команду»** (о ней ниже).

### 4. Как человек попадает в «Организация → Команда»

Здесь три входа:

#### 4.1. Приглашение по ссылке

- В разделе **«Организация → Команда»** есть кнопка:
  - **«Пригласить в команду»** → диалог:
    - Выбор роли в организации (дизайнер, разработчик, PM, наблюдатель).
    - Выбор уровня доступа (админ, редактор, только просмотр и т.п.).
((руль доступа выбирать не нужно человек при регистрации после получения ссылки сам выберет для себя роль))
  - Нажимаешь → генерируется **пригласительная ссылка**.

- Дальше сценарии:
  - Если человек **не зарегистрирован**:
    - Открывает ссылку → форма регистрации → после регистрации его **автоматически добавляет** в эту организацию с выбранной ролью.
  - Если человек **уже зарегистрирован**:
    - Открывает ссылку → видит «Вас приглашают в организацию X как [роль]» → нажимает «Принять» → появляется в списке команды.

#### 4.2. Приглашение по email

- В том же диалоге есть режим **«Пригласить по email»**:
  - Вводишь email, выбираешь роль в организации.
  - Если такой пользователь уже есть:
    - Он получает уведомление внутри системы + письмо.
  - Если нет:
    - Уходит письмо с приглашением и ссылкой на регистрацию.
- После подтверждения → пользователь **добавляется в твою организацию**.

#### 4.3. Приглашение из каталога «Исполнители»

- На карточке исполнителя в разделе **«Исполнители»**:
  - Кнопка **«Пригласить в команду»**.
((также должна быть возможность пообщаться с этим исполнителем в чате))
- При нажатии:
  - Если ты владелец/админ нескольких организаций – сначала выбираешь, **в какую**.
  - Выбираешь **роль в команде** (дизайнер, dev и т.п.).
((руль выбирать не нужно так как если ты приглашаешь исполнителя который уже зарегистрирован то роль у него уже в принципе есть))
  - Отправляется **организационное приглашение** этому исполнителю.
  - У него в личном кабинете:
    - Уведомление: «Организация X приглашает вас в команду как [роль]».
    - Кнопки «Принять» / «Отклонить».
((кроме кнопки принятия отклонить должен быть диалог то есть человек может поинтересоваться, пообщаться спросить зачем почему. Ещё он может перейти в сам проект предварительно с ним ознакомиться и понять сможет ли он выполнит ту роль которую от него просят. При этом он не может смотреть задачи которые есть в проекте только лишь телефон проект его описание его стадию и вообще о чем этот проект ну то есть изучить его))
- После принятия:
  - Исполнитель попадает в **«Организация → Команда»**.
  - Может быть помечен как **внешний** (если вы хотите отличать штат от фрилансеров) или обычный член команды.

### 5. Как из проекта / задачи выбирать исполнителя

Здесь две ключевые идеи:

1. **Назначаем только людей, которые уже в моей организации** (по умолчанию).
2. **Из проекта можно инициировать приглашение**, если нужного человека ещё нет.

#### 5.1. UI выбора исполнителя в проекте

В карточке проекта:

- Раздел **«Команда проекта»**:
  - Кнопка **«Добавить участника»**.
  - Открывается комбобокс / селект с поиском:
    - По умолчанию список **участников организации** (люди из «Организация → Команда»).
    - Над списком строка поиска по имени.

В задаче:

- Поле **«Ответственный»** (single select) и, возможно, **«Исполнители»** (multi-select).
- При фокусе на поле:
  - Появляется тот же комбобокс:
    - **По умолчанию** – члены команды проекта / организации.
    - Можно печатать имя и фильтровать.

#### 5.2. Что делать, если человека нет в команде

В комбобоксе, если нет совпадений:

- Появляются опции:

  - **«Пригласить по email в организацию и в проект»**
    - Вводишь email → создаётся приглашение в организацию **сразу с привязкой к этому проекту**.
  - **«Найти в каталоге исполнителей»**
    - Открывается модальное окно с мини-версией каталога выбранной специализации.
    - На карточках – кнопка **«Пригласить в команду и в проект»**.

Логика под капотом:

- При выборе человека:
  - Если он уже член организации, но не в проекте:
    - Добавляем в **команду проекта**.
  - Если он пока не член организации:
    - Создаём приглашение в организацию.
    - После принятия – автоматически добавляем его в выбранный проект/задачу (или просим подтвердить).

### 6. Кнопка «Пригласить в команду» на карточке исполнителя

**Раздел «Исполнители» → карточка исполнителя:**

- Кнопки:
  - **«Посмотреть профиль»**
  - **«Пригласить в команду»**

**Фlow:**

1. Нажимаешь «Пригласить в команду».
2. Открывается модальное окно:
   - Если у пользователя нет организации:
     - Предлагаем: «Сначала создайте организацию, чтобы приглашать людей».
   - Если есть:
     - Поля:
       - **Организация** (если их несколько).
       - **Роль в организации** (например, дизайнер, разработчик, аналитик).
       - Галочка: **«Сразу добавить в проект»** + селект проекта из списка активных проектов.
3. После подтверждения:
   - Исполнитель получает уведомление.
   - После принятия:
     - Добавляется в **«Организация → Команда»**.
     - (Если выбрали проект) – сразу в **команду проекта**.

### 7. Итоговая цепочка, как ты просил – от регистрации до назначения в задачу

**Сценарий А: Исполнитель сам заходит на платформу**

1. Регистрируется → выбирает «Я исполнитель» → настраивает профиль → включает «Показывать в каталоге».
2. Попадает в **каталог исполнителей**.
3. Организатор находит его в «Исполнители» → жмёт «Пригласить в команду».
4. Исполнитель принимает приглашение → появляется в **«Организация → Команда»**.
5. Менеджер заходит в проект → в селекте «Добавить участника / Ответственный» выбирает этого человека из списка.
6. Теперь в задачах по проекту его можно ставить ответственным.

**Сценарий B: Ты уже знаешь исполнителя лично**

1. Ты создаёшь организацию.
2. В разделе «Организация → Команда» жмёшь:
   - «Пригласить по ссылке» или «Пригласить по email».
3. Исполнитель по ссылке регистрируется как пользователь (если его ещё нет).
4. При регистрации ему можно:
   - Либо сразу предложить создать **профиль исполнителя** (чтобы он появился в каталоге).
5. После принятия приглашения он:
   - В списке команды.
   - Доступен для выбора в проектах/задачах.
((тут я выбираю сценарий B:  пользователь которого пригласили по имейл должен сразу попадать в организацию и становиться частью команды при этом далее он сам выбирает будет ли он попадать в каталог. Это уже его дальнейший выбор ))
