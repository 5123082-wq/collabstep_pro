# Промпты для агентной работы по интеграции «Исполнителей»

Ниже шаблоны промптов для отдельных агентов. Каждый промпт отдаётся целиком, без сокращений.

---

## 1) Агент‑аудитор (инвентаризация текущей реализации)

**Задача:** проанализировать текущее состояние раздела «Исполнители» в коде и данных, составить карту пробелов.

**Промпт:**
```
Ты — агент‑аудитор. Проанализируй текущее состояние раздела «Исполнители» в репозитории.

Контекст:
- Раздел включает специалистов, команды, подрядчиков.
- Нужно понять, что уже реализовано и где пробелы.

Что нужно сделать:
1) Найти существующие модели данных, API и UI, связанные с исполнителями.
2) Составить таблицу: «Функция → Где в коде → Статус (готово/частично/нет) → Риски».
3) Отдельно перечислить, чего не хватает для:
   - команд исполнителей,
   - подрядчиков (компании),
   - RFQ/КП/этапов оплаты.

Формат результата:
- Короткий summary.
- Таблица инвентаризации.
- Список пробелов по группам.
```

---

## 2) Агент‑архитектор (модель данных + API)

**Задача:** спроектировать модель данных и API для команд и подрядчиков, с учётом интеграции в проекты/документы/финансы.

**Промпт:**
```
Ты — агент‑архитектор. Спроектируй модель данных и API для раздела «Исполнители» с учётом команд и подрядчиков.

Контекст:
- Уже есть performer_profile для специалистов.
- Нужно добавить команды и подрядчиков + контур RFQ/КП/SLA/приёмки.

Что нужно сделать:
1) Предложить новые сущности и связи (ER‑описание текстом).
2) Описать ключевые поля (минимальный набор для MVP).
3) Составить список API‑эндпоинтов (CRUD + поиск + workflow).
4) Указать, как это связывается с:
   - проектами и задачами,
   - документами/контрактами,
   - финансами/эскроу.

Формат результата:
- Схема сущностей (список + связи).
- API‑список с краткими описаниями.
- MVP‑набор полей.
```

---

## 3) Агент‑UX (пользовательские сценарии и интерфейсы)

**Задача:** описать пользовательские сценарии и экраны для команд и подрядчиков.

**Промпт:**
```
Ты — агент‑UX. Опиши пользовательские сценарии и интерфейсы для команд исполнителей и подрядчиков.

Контекст:
- Платформа включает проекты/задачи/документы/финансы.
- В разделе «Исполнители» будут команды и подрядчики.

Что нужно сделать:
1) Опиши ключевые сценарии:
   - создание профиля команды,
   - создание профиля подрядчика,
   - поиск и выбор исполнителя,
   - RFQ/бриф,
   - принятие КП,
   - запуск сделки в проект.
2) Для каждого сценария укажи набор экранов и ключевые поля.
3) Отдельно опиши UX для заказчиков и исполнителей.

Формат результата:
- Список сценариев + шаги.
- Список экранов.
- Перечень ключевых полей.
```

---

## 4) Агент‑интегратор (дорожная карта внедрения)

**Задача:** разбить внедрение на этапы с приоритетами и зависимостями.

**Промпт:**
```
Ты — агент‑интегратор. Построй дорожную карту внедрения раздела «Исполнители» (команды и подрядчики).

Контекст:
- Нужно MVP, затем расширение (trust/споры/SLA).

Что нужно сделать:
1) Разбей работы на 3 этапа (MVP / сделки / доверие и операции).
2) Для каждого этапа опиши задачи, зависимости и критерии готовности.
3) Укажи быстрые победы и долгий цикл.

Формат результата:
- Таблица этапов (этап, задачи, зависимости, DoD).
- Короткий список quick wins и long cycle.
```
