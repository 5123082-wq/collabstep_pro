Краткое описание пользовательской поддержки, каналов обращений и UI-разделов.

## 1) Назначение

Документ описывает, как пользователи получают поддержку и где находятся соответствующие разделы в интерфейсе.

## 2) Текущее состояние

- Есть UI-разделы поддержки (help, tickets, chat).
- Выбранная модель поддержки: тикеты + чат.

## 3) Пользовательские маршруты (UI)

- `/support/help` — база знаний/справка
- `/support/tickets` — обращения/тикеты
- `/support/chat` — чат поддержки

## 5) Модель данных (draft)

### Ticket

Минимальные поля:

- `id` — идентификатор
- `created_at`, `updated_at`
- `created_by_user_id`
- `assigned_to_user_id` (опционально)
- `status` — статус обращения
- `priority` — приоритет
- `subject` — тема
- `description` — описание
- `tags[]` — теги

### Chat message

Минимальные поля:

- `id` — идентификатор
- `created_at`
- `conversation_id`
- `author_user_id`
- `message` — текст сообщения
- `attachments[]` — вложения (опционально)

## 6) Статусы и приоритеты (draft)

### Статусы тикетов

- `new` — новое обращение
- `in_progress` — в работе
- `waiting_user` — требуется ответ пользователя
- `resolved` — решено
- `closed` — закрыто

### Приоритеты

- `low`, `medium`, `high`, `urgent`
