# Маркетплейс — Корзина

## Назначение

Корзина собирает выбранные шаблоны для оформления заказа. В текущей версии корзина работает локально и не синхронизируется с сервером.

## Навигация

- `/market/cart` — корзина

## Текущее состояние (по UI и коду)

- Состояние корзины хранится в `useMarketplaceStore` (Zustand).
- Данные товаров подставляются из мок‑набора шаблонов через `enrichCartItems`.
- Количество нельзя опустить ниже 1.
- Поле промокода не влияет на расчёт (UI‑заглушка).
- Итоговая сумма считается только для `pricingType: 'paid'`.
- Кнопка «Оформить заказ» очищает корзину и показывает alert (демо‑сценарий).

## Основные сценарии

1. Добавить шаблон в корзину из каталога или избранного.
2. Изменить количество позиций.
3. Удалить позицию или очистить корзину.
4. Запустить демо‑оформление заказа.

## Доступность по ролям

- Раздел доступен ролям `FOUNDER`, `PM`, `ADMIN`, `SPECIALIST`, `CONTRACTOR`.

## Взаимодействия с другими разделами

- **Каталог шаблонов:** источник товаров.
- **Избранное:** альтернативный источник добавления.
- **Мои заказы:** планируемая история оформленных покупок.

## Ограничения и планируемое

- Нет интеграции с платёжным провайдером и сервисом заказов.
- Корзина не сохраняется между сессиями.
- Подписочные и бесплатные шаблоны не участвуют в расчёте суммы (пока).
