# Маркетплейс — Готовые проекты

**Статус:** partial
**Владелец:** product/engineering
**Создан:** 2026-01-07
**Последнее обновление:** 2026-01-07

## Назначение

Витрина «Готовые проекты» должна показывать опубликованные проекты, которые можно приобрести или использовать как основу. Источником таких проектов является публикация из PM‑раздела.

## Навигация

- `/market/projects` — готовые проекты

## Текущее состояние (по UI и коду)

- Страница `/market/projects` показывает заглушку «в разработке».
- В API реализован листинг проекта `/api/pm/projects/:id/listings` со статусами `draft`, `published`, `rejected`.
- Статус листинга отображается в карточке проекта (блок «Связки»).

## Планируемые сценарии

1. Просмотр витрины готовых проектов.
2. Переход в карточку проекта и ознакомление с составом.
3. Покупка/запрос проекта и переход к оформлению.

## Спорные статусы и роли

- **`rejected`** — листинг отклонён (модерация). Решение принимает администратор/модератор.

## Доступность по ролям

- Раздел доступен ролям `FOUNDER`, `PM`, `ADMIN`, `SPECIALIST`, `CONTRACTOR`.

## Взаимодействия с другими разделами

- **Проекты и задачи:** источник листинга.
- **Опубликовать:** мастер публикации готовых проектов.
- **Финансы:** оформление оплаты (планируется).

## Ограничения и планируемое

- Витрина не связана с листингами напрямую, требуется интеграция.
