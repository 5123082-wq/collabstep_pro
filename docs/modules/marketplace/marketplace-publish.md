# Маркетплейс — Опубликовать

**Статус:** partial
**Владелец:** product/engineering
**Создан:** 2026-01-07
**Последнее обновление:** 2026-01-07

## Назначение

Раздел «Опубликовать» должен стать мастером публикации проектов, шаблонов и пакетов услуг в маркетплейсе.

## Навигация

- `/market/publish` — опубликовать

## Текущее состояние (по UI и коду)

- Страница `/market/publish` отображает заглушку «в разработке».
- Листинг проекта можно создать через API `/api/pm/projects/:id/listings` (флаг `PM_PROJECT_CARD`).
- Статусы листинга: `draft`, `published`, `rejected`.

## Основные сценарии (план)

1. Запуск мастера публикации.
2. Выбор типа публикации (проект/шаблон/услуга).
3. Настройка карточки и условий публикации.
4. Отправка на модерацию и публикация.

## Спорные статусы и роли

- **`rejected`** — листинг отклонён (модерация). Решение принимает администратор/модератор.

## Доступность по ролям

- Создание листинга проекта доступно ролям owner/admin проекта.
- Модерация и публикация — для администраторов/модераторов.

## Взаимодействия с другими разделами

- **Проекты и задачи:** источник данных для публикации.
- **Готовые проекты:** будущая витрина опубликованных листингов.

## Ограничения и планируемое

- UI‑мастер публикации не реализован.
