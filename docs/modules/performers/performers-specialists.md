# Исполнители — Специалисты

**Последнее обновление:** 2026-01-15

## Назначение

Раздел «Специалисты» показывает каталог публичных профилей исполнителей, позволяет фильтровать экспертов и приглашать их в команды.

## Навигация

- `/performers/specialists` — каталог специалистов
- `/p/:handle` — публичная визитка

## Текущее состояние (по UI и коду)

- Каталог использует публичные профили из `performer_profile` и отображает рейтинги.
- Поиск, фильтры и сортировки работают на фронтенде, есть быстрые фильтры и блок «Все фильтры».
- Карточка специалиста поддерживает действия: «Пригласить», «Запросить интервью», «Открыть визитку».
- Публичная визитка отображает реальные данные профиля, портфолио, кейсы и отзывы.

## Модель данных (кратко)

Профиль исполнителя (`performer_profile`):
- `specialization` — специализация/роль
- `skills` — массив навыков
- `bio` — описание
- `rate` — ставка (число)
- `employmentType` — `fulltime` | `parttime` | `contract`
- `location`, `timezone`
- `isPublic` — видимость в каталоге

Схема карточки специалиста в каталоге строится из профиля и дополняется UI‑полями (рейтинг, языки, формат работы).

## Фильтры каталога

UI:
- Быстрые фильтры: роль, формат работы, сортировка, поиск.
- «Все фильтры»: навыки, язык, ставка.

Поддерживаются параметры URL:
- `q` — поиск
- `role` — роль
- `skills` — список навыков через запятую
- `lang` — язык
- `format` — `remote` | `office` | `hybrid`
- `rateMin`, `rateMax` — фильтр ставки
- `sort` — `rating` | `cost` | `new`
- `page` — страница

## Основные сценарии

1. Найти специалиста через фильтры и поиск.
2. Открыть визитку специалиста.
3. Пригласить специалиста в организацию/проект.
4. Создать и опубликовать собственную карточку.

## Создание

1. Перейти в `/settings/performer` или `/onboarding/create-profile`.
2. Заполнить специализацию, навыки, био, ставку и тип занятости.
3. При необходимости выбрать тип пользователя «Исполнитель» в настройках профиля.
4. Включить `isPublic`, чтобы карточка появилась в каталоге.
5. Сохранить профиль.

## Спорные статусы и роли

- **`isPublic=false`** — профиль доступен только владельцу.
- Приглашение в организацию/проект может создать только owner/admin.

## Доступность по ролям

- Каталог доступен всем авторизованным пользователям.
- Редактирование профиля доступно только владельцу профиля.

## API (MVP)

- `GET /api/performers?specialization=&limit=&offset=` — список публичных профилей
- `GET /api/performers/:userId` — публичный профиль
- `GET /api/me/performer-profile` — профиль текущего пользователя
- `POST /api/me/performer-profile` — создать/обновить профиль
- `PATCH /api/me/performer-profile/visibility` — изменить публичность
- `GET /api/me/performer-profile/portfolio` — список работ
- `POST /api/me/performer-profile/portfolio` — добавить работу
- `DELETE /api/me/performer-profile/portfolio?id=` — удалить работу
- `POST /api/me/performer-profile/portfolio/upload-url` — получить upload-url для файла
- `GET /api/me/performer-profile/cases` — список кейсов
- `POST /api/me/performer-profile/cases` — добавить кейс
- `DELETE /api/me/performer-profile/cases?id=` — удалить кейс
- `GET /api/performers/:userId/rating-summary` — сводный рейтинг
- `GET /api/performers/:userId/ratings` — список отзывов
- `POST /api/performers/:userId/ratings` — оставить рейтинг (требуется `projectId`)
- `POST /api/performers/:userId/invite-to-organization` — пригласить в организацию/проект

## Взаимодействия с другими разделами

- **Проекты и задачи:** приглашение исполнителей в команды проектов.
- **Организация:** инвайты в организацию.

## Ограничения и планируемое

- **Условия рейтинга:** рейтинг доступен только для участников проекта со статусом `completed` | `archived`, рейтить могут участники проекта, не `viewer`; обе стороны должны быть участниками проекта.
- **Портфолио (MVP/следующий этап):**
  - MVP: загрузка файлов через Vercel Blob, `file_url` хранит blob‑ссылку.
  - Следующий этап: подключение внешних источников/ссылок (Behance, Dribbble, Notion, Drive и т.д.).
