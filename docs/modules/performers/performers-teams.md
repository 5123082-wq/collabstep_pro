## Назначение

Раздел «Команды» предназначен для подбора команд исполнителей — групп специалистов, объединённых на платформе, которые берут комплексные задачи и работают как единая команда.

## Навигация

- `/performers/teams` — команды

## Текущее состояние (по UI и коду)

- Страница отображает сценарные действия без реального каталога.
- Модель команд ведётся в PM‑модуле (участники проекта/организации).
- Отдельный профиль команды пока не реализован.

## Планируемые сценарии

1. Просмотр каталога команд.
2. Запрос предложения от команды.
3. Сравнение команд по специализации, ставке и опыту.

## Спорные статусы и роли

- **Команда vs участник:** команда — профиль группы, участник — конкретный исполнитель.
- Роли внутри команды планируются как owner/manager/member.

## Доступность по ролям

- Просмотр каталога — всем авторизованным пользователям.
- Приглашение в команду — owner/admin организации.

## Взаимодействия с другими разделами

- **Проекты и задачи:** роли участников и доступы к проектам.
- **Организация:** управление членством и приглашениями.

## Ограничения и планируемое

- Каталог команд пока не реализован.

## Дорожная карта (на базе исследований)

### Команды исполнителей
- Профиль команды с ролями и составом.
- Поиск по набору компетенций (team‑match).
- Воркрум сделки: проект + задачи + документы + этапы оплаты.
- Метрики репутации по факту исполнения.
