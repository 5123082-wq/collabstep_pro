# Исполнители — Вакансии и задачи

**Статус:** partial
**Владелец:** product/engineering
**Создан:** 2026-01-07
**Последнее обновление:** 2026-01-07

## Назначение

Раздел «Вакансии и задачи» собирает запросы на исполнителей и проектные задачи. Здесь планируется публиковать вакансии и принимать отклики.

## Навигация

- `/performers/vacancies` — список вакансий
- `/performers/vacancies/:id` — карточка вакансии

## Текущее состояние (по UI и коду)

- Каталог вакансий отображается, но данные пока пустые.
- Карточка вакансии использует компонент `VacancyDetail`, но API не подключен.
- Фильтры и сортировка реализованы на фронтенде.

## Фильтры каталога

Параметры URL:
- `q` — поиск
- `role` — роль
- `level` — `Junior` | `Middle` | `Senior`
- `employment` — `project` | `part-time` | `full-time`
- `format` — `remote` | `office` | `hybrid`
- `reward` — `rate` | `salary` | `equity`
- `lang` — язык
- `page` — страница

## Основные сценарии

1. Просмотреть список вакансий и отфильтровать по параметрам.
2. Открыть карточку вакансии и изучить требования.
3. Отправить отклик (планируется).

## Создание

Создание вакансий планируется в разделе «Мои вакансии». Контекстных действий для кнопки «Создать» пока нет.

## Спорные статусы и роли

- Статусы вакансий и откликов будут зафиксированы после внедрения API.
- Публикацию вакансий должны контролировать owner/admin организаций.

## Доступность по ролям

- Просмотр вакансий доступен всем авторизованным пользователям.

## Взаимодействия с другими разделами

- **Проекты и задачи:** вакансии связаны с задачами и ролями в проектах.
- **Исполнители — Отклики:** место обработки откликов.

## Ограничения и планируемое

- Нет данных и API для вакансий.
