## Введение

AI-агенты — это автономные AI-агенты для автоматизации задач и процессов в проектах. Планируется реализация в Stage N.

## Структура агента

```ts
interface AIAgent {
  id: string;
  name: string;
  type: AIAgentType; // 'assistant' | 'reviewer' | 'reminder' | 'summarizer'
  scope: AIAgentScope; // 'personal' | 'team' | 'public'
  projectId?: string; // Для агентов проекта
  triggers: AIAgentTrigger[]; // Триггеры активации
  actions: AIAgentAction[]; // Действия агента
  enabled: boolean;
  createdAt: string;
  updatedAt: string;
}
```

## Типы агентов

- **assistant** — помощник для ответов на вопросы
- **reviewer** — ревьюер задач и кода
- **reminder** — напоминания о дедлайнах
- **summarizer** — суммирование комментариев и обсуждений

## Триггеры агентов

Агенты активируются при определенных событиях:

- **task.created** — при создании задачи
- **task.updated** — при обновлении задачи
- **comment.added** — при добавлении комментария
- **deadline.approaching** — при приближении дедлайна
- **mention** — при упоминании агента в чате

## Создание агента

1. Пользователь открывает настройки проекта
2. Переходит в раздел "AI-агенты"
3. Нажимает "Создать агент"
4. Заполняет форму:
   - Название агента
   - Тип агента
   - Область видимости (personal/team/public)
   - Триггеры активации
   - Действия агента
5. Сохраняет агента

## Использование агентов

### В чате проекта

Агенты могут быть упомянуты в чате:
```
@ai-assistant Как лучше организовать задачи?
```

### Автоматические действия

Агенты выполняют действия автоматически при срабатывании триггеров:
- Создание задач
- Обновление статусов
- Отправка уведомлений
- Генерация контента
