## Введение

AI-ассистент для проектов — это интеллектуальный помощник, который помогает пользователям изучать и использовать платформу Collabverse. Ассистент работает на основе RAG (Retrieval-Augmented Generation) технологии и использует документацию для точных ответов.

## Реализованные функции (Stage M)

- ✅ Генерация описания задачи
- ✅ Суммирование комментариев
- ✅ AI агенты в чате (упоминание `@ai-assistant`)
- ✅ Напоминания о дедлайнах

## Использование в чате

### Упоминание @ai-assistant

1. Откройте чат проекта
2. Упомяните `@ai-assistant` в сообщении
3. Задайте вопрос или попросите помощи
4. AI-ассистент ответит на основе контекста проекта

**Пример:**
```
@ai-assistant Как лучше организовать задачи для этого проекта?
```

### Команды в чате

- `/ai вопрос` — задать вопрос AI-ассистенту
- `@ai-assistant` — упомянуть ассистента в сообщении

## Генерация описаний задач

### Процесс

1. Пользователь создает новую задачу
2. Вводит название задачи
3. Нажимает "✨ Сгенерировать описание"
4. AI создает структурированное описание задачи в markdown
5. Пользователь может редактировать описание перед сохранением

### Пример

**Название задачи:** "Создать REST API для пользователей"

**Сгенерированное описание:**
```md
## Цель
Создать REST API для управления пользователями с поддержкой CRUD операций.

## Задачи
- [ ] Спроектировать структуру API endpoints
- [ ] Реализовать endpoints для создания, чтения, обновления и удаления пользователей
- [ ] Добавить валидацию данных
- [ ] Написать тесты для API
- [ ] Документировать API

## Критерии готовности
- Все endpoints работают корректно
- Валидация данных реализована
- Тесты покрывают основные сценарии
- API задокументирован
```

## Суммирование комментариев

### Процесс

1. Пользователь открывает задачу с множеством комментариев
2. Нажимает "Суммировать комментарии"
3. AI создает краткую сводку обсуждений
4. Пользователь видит основные моменты обсуждения

### Пример

**Комментарии:**
- "Нужно добавить валидацию email"
- "Также проверить формат телефона"
- "Можно использовать библиотеку validator.js"
- "Согласен, добавим в следующей итерации"

**Суммарка:**
```markdown
## Основные моменты обсуждения

- Решено добавить валидацию email и телефона
- Предложено использовать библиотеку validator.js
- Планируется реализация в следующей итерации
```

## Напоминания о дедлайнах

AI создает персонализированные напоминания о приближающихся дедлайнах задач.

### Процесс

1. AI анализирует задачи с дедлайнами
2. Определяет задачи с приближающимися дедлайнами
3. Создает персонализированные напоминания для исполнителей
4. Отправляет уведомления через систему уведомлений

## Контекстные подсказки

AI-ассистент предлагает контекстные подсказки в зависимости от текущей страницы:

- На странице проекта — подсказки по управлению проектом
- На странице задачи — подсказки по работе с задачей
- В чате — подсказки по использованию функций платформы

## API Endpoints

### POST /api/ai/generate-description

Генерация описания задачи

**Request:**
```json
{
  "taskTitle": "Создать REST API",
  "projectId": "optional-project-id"
}
```

**Response:**
```json
{
  "description": "Структурированное описание задачи в markdown"
}
```

### POST /api/ai/summarize-comments

Суммирование комментариев задачи

**Request:**
```json
{
  "taskId": "task-id",
  "projectId": "project-id"
}
```

**Response:**
```json
{
  "summary": "Краткая сводка комментариев в markdown"
}
```
